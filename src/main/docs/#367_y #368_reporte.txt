CÓDIGO CLAVE (Ejemplo del manejo de error y creación de Excel):

// 1. Refactorización para Empty State (Aplica para CSV y Excel)
if (totales == null || totales.getTotalViajes() == 0) {
    throw new RuntimeException("EMPTY_STATE");
}

// 2. Creación del Excel en memoria
public java.io.ByteArrayInputStream generarExcelOperativo(LocalDate fechaInicio, LocalDate fechaFin) throws java.io.IOException {
    // ... obtención de datos ...
    
    try (Workbook workbook = new XSSFWorkbook(); 
         ByteArrayOutputStream out = new ByteArrayOutputStream()) {
         
        Sheet sheet = workbook.createSheet("Reporte Operativo");
        
        // Estilos para la cabecera
        CellStyle headerStyle = workbook.createCellStyle();
        Font font = workbook.createFont();
        font.setBold(true);
        headerStyle.setFont(font);
        
        // Inserción de datos
        Row headerRow = sheet.createRow(0);
        headerRow.createCell(0).setCellValue("Métrica");
        headerRow.createCell(1).setCellValue("Valor");
        
        sheet.createRow(1).createCell(0).setCellValue("Total de Viajes");
        sheet.getRow(1).createCell(1).setCellValue(totales.getTotalViajes());
        
        // ... (resto de las filas) ...
        
        workbook.write(out);
        return new ByteArrayInputStream(out.toByteArray());
    }
}
EXPLICACIÓN:

XSSFWorkbook: Crea un archivo de Excel formato .xlsx.

ByteArrayOutputStream: Mantiene el archivo en la memoria RAM (como un flujo de bytes) sin necesidad de guardarlo físicamente en el disco duro del servidor.

throw new RuntimeException("EMPTY_STATE"): Estandariza la señal que avisa que los filtros no trajeron datos.

POR QUE:

Mantener la lógica en ReporteService.java evita crear clases innecesarias, reciclando las consultas a la base de datos que ya existían.

Escribir en memoria (ByteArray) es la forma más rápida y segura de enviar un archivo por red.

3. CLASE MODIFICADA: ReporteController.java
═══════════════════════════════════════════════════════════════════════════════

Ubicación: src/main/java/com/logitrack/sistema_logistica/controller/ReporteController.java

QUE SE AGREGÓ:
Nuevos endpoints para descargar el Excel y refactorización de los endpoints CSV para capturar el EMPTY_STATE y configurar las cabeceras HTTP correctas.

CÓDIGO CLAVE:

Java
@GetMapping("/operativo/exportar/excel")
public ResponseEntity<?> exportarOperativoExcel(
        @RequestParam(required = false) @DateTimeFormat(iso = DateTimeFormat.ISO.DATE) LocalDate fechaInicio,
        @RequestParam(required = false) @DateTimeFormat(iso = DateTimeFormat.ISO.DATE) LocalDate fechaFin) {
    try {
        java.io.ByteArrayInputStream stream = reporteService.generarExcelOperativo(fechaInicio, fechaFin);
        
        HttpHeaders headers = new HttpHeaders();
        headers.add("Content-Disposition", "attachment; filename=\"Logitrack_Operativo.xlsx\"");
        headers.add("Access-Control-Expose-Headers", "Content-Disposition");

        return ResponseEntity.ok()
                .headers(headers)
                .contentType(MediaType.parseMediaType("application/vnd.openxmlformats-officedocument.spreadsheetml.sheet"))
                .body(new InputStreamResource(stream));

    } catch (RuntimeException | java.io.IOException e) {
        if ("EMPTY_STATE".equals(e.getMessage())) {
            return ResponseEntity.noContent().build(); // Devuelve 204
        }
        return ResponseEntity.badRequest().body(e.getMessage());
    }
}
EXPLICACIÓN LÍNEA POR LÍNEA:

Content-Disposition: attachment: Le indica al navegador que debe descargar el archivo en lugar de intentar mostrarlo en pantalla.

Access-Control-Expose-Headers: Fundamental para que el frontend pueda leer el nombre del archivo sugerido.

MediaType...spreadsheetml.sheet: Es el tipo MIME oficial que le dice a la computadora "Esto es un archivo de Excel".

if ("EMPTY_STATE"...) return ResponseEntity.noContent().build();: Si el servicio avisa que no hay datos, atajamos el error y respondemos con un HTTP 204.

POR QUE:

Separa claramente la ruta de datos JSON (/operativo) de la ruta de descarga (/operativo/exportar/excel).

El frontend puede detectar fácilmente el 204 y mostrar una alerta en lugar de descargar un archivo corrupto.

═══════════════════════════════════════════════════════════════════════════════
FLUJO COMPLETO DE EJECUCIÓN (Ejemplo Excel Operativo)
═══════════════════════════════════════════════════════════════════════════════

Usuario hace clic en "Exportar Excel":
GET http://localhost:8080/api/reportes/operativo/exportar/excel?fechaInicio=2026-06-01&fechaFin=2026-06-30

ReporteController recibe la petición.

Llama a reporteService.generarExcelOperativo(...).

El servicio busca los datos. Si no hay datos, lanza la excepción "EMPTY_STATE". El controlador la atrapa y devuelve 204 No Content (Fin del flujo).

Si hay datos, el servicio crea un archivo Excel en memoria usando Apache POI y lo llena celda por celda.

El servicio devuelve un flujo de bytes al controlador.

El controlador arma la respuesta HTTP adjuntando el archivo con las cabeceras attachment y el tipo MIME de Excel.

El navegador del usuario inicia la descarga de Logitrack_Operativo.xlsx.

Autorízate con el Token.

Busca /api/reportes/operativo/exportar/excel, ingresa fechas y presiona "Execute".

PARA PROBAR EL EMPTY STATE:

Ingresa un rango de fechas en el futuro (ej. 2030-01-01). El servidor no descargará nada y devolverá código de estado 204 No Content.
